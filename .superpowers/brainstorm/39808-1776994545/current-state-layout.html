Current State Page — Layout Options

How should the polished Current State page be organized? Each option keeps the same data; what changes is the hierarchy and flow.

A

Narrative-First

A 2–3 sentence plain-English summary at the top ("The market is in an elevated regime. VIX is at 18.9, rising slightly. Transition risk is low at 0.5%.") followed by the regime pill + gauge + trend in a row, then the driver chart below.

Best for: users who want immediate interpretation, not just numbers. More product-like.

B

Metrics-First with Narrative Below

Regime pill + gauge + VIX/trend metrics in the top row (as today), then a narrative summary paragraph below that, then the driver chart. The narrative becomes secondary context rather than the lead.

Best for: users who want the numbers first and explanation second. Closer to current layout.

C

Two-Column Split

Left column: regime pill + narrative summary + trend chip + "why did it change" delta. Right column: gauge + VIX card + mode badge. Driver chart spans full width below.

Best for: dense information density with clear left/right hierarchy. More dashboard-like.

Shared across all options

  • Mode badge renamed to NEAR-LIVE DATA (live) or DEMO MODE (demo)
  • Plain-English narrative auto-generated from regime + risk + trend values
  • "No feature driver data" fallback replaced with a proper explanation
  • Regime class probabilities shown if available (calm 12% / elevated 81% / turbulent 7%)
  • Delta panel: how risk changed since last refresh
